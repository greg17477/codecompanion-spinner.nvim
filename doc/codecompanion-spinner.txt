1. Overview                                   *codecompanion-spinner-overview*

Elegant, state-aware status feedback for CodeCompanion.nvim
<https://github.com/olimorris/codecompanion.nvim>.

This plugin provides a non-intrusive floating spinner that tracks the
lifecycle of AI requests, tool executions, and user interactions within your
CodeCompanion chat buffers.

==============================================================================
2. Features                                   *codecompanion-spinner-features*

- Lifecycle Tracking: Visual feedback for thinking, receiving (streaming),
  tool execution, and diff generation.
- State-Aware Messages: Contextual indicators for "Awaiting Approval",
  "Diff Attached", and more.
- Multi-Chat Support: Concurrent requests across different chat buffers are
  managed independently.
- Zero-Config by Default: Works out of the box with sensible defaults.
- Smart Positioning: Heuristic floating window placement relative to the
  chat buffer.

3. Installation                           *codecompanion-spinner-installation*

Install via your preferred package manager and register the extension in your
|codecompanion| setup:

Example using lazy.nvim:
>lua
    {
      "olimorris/codecompanion.nvim",
      dependencies = {
        "nvim-lua/plenary.nvim",
        "greg17477/codecompanion-spinner.nvim",
      },
      opts = {
        -- ... your existing config ...
        extensions = {
          spinner = {}, -- Load the extension with default settings
        },
      },
    }
<

4. Configuration                         *codecompanion-spinner-configuration*

While the spinner works without configuration, you can customize its behavior:

>lua
    extensions = {
      spinner = {
        log_level = "info",
        spinner_symbols = { "⠋", "⠙", "⠹", "⠸", "⠼", "⠴", "⠦", "⠧", "⠇", "⠏" },
        done_timer = 2000, -- Duration (ms) to show the "Done!" message
        timer_interval = 200, -- Animation speed
        messages = {
          thinking = "thinking",
          receiving = "receiving",
          tool_running = "tool running",
          tool_finished = "tool finished",
          tool_processing = "tool processing",
          awaiting_approval = "󱗿 Awaiting approval",
          diff_attached = "󰙶 Diff attached",
          done = "󰄬 Done!",
          stopped = "stopped",
        },
        symbols = {
          thinking = nil,
          receiving = nil,
          tool_running = nil,
          tool_finished = "󰄬",
          tool_processing = "󱗿",
          awaiting_approval = "󱗿",
          diff_attached = "󰙶",
          done = "󰄬",
          stopped = "󰓛",
        },
        window = {
          width = 20,
          height = 1,
          row = -2,    -- Offset from bottom of the chat window
          col = -1,    -- Offset from right edge
          padding = 1, -- Right padding
          winblend = 0,
          zindex = 200,
          border = "none",
          style = "minimal",
          focusable = false,
          noautocmd = true,
          winhl = nil, -- e.g., 'Normal:Comment,NormalNC:Comment'
        },
        highlights = {
          spinner = "DiagnosticError",
          thinking = "DiagnosticHint",
          receiving = "DiagnosticInfo",
          awaiting_approval = "DiagnosticWarn",
          diff_attached = "DiagnosticWarn",
          tool_running = "DiagnosticHint",
          tool_finished = "DiagnosticOk",
          tool_processing = "DiagnosticHint",
          done = "DiagnosticOk",
          -- Symbol-specific highlights
          thinking_symbol = "DiagnosticError",
          receiving_symbol = "DiagnosticError",
          tool_running_symbol = "DiagnosticError",
          tool_finished_symbol = "DiagnosticOk",
          tool_processing_symbol = "DiagnosticWarn",
          awaiting_approval_symbol = "DiagnosticWarn",
          diff_attached_symbol = "DiagnosticWarn",
          done_symbol = "DiagnosticOk",
          stopped_symbol = "DiagnosticError",
        },
      },
    },

5. Highlights                               *codecompanion-spinner-highlights*

The spinner uses the following highlight groups by default:

Group                           Default           Description ~
spinner                         DiagnosticError   The animated symbol
thinking                        DiagnosticHint    Text while AI is processing
receiving                       DiagnosticInfo    Text while AI is streaming
awaiting_approval               DiagnosticWarn    When a tool requires approval
diff_attached                   DiagnosticWarn    When a diff is ready
done                            DiagnosticOk      Final success state

Symbol-specific highlights (e.g., `thinking_symbol`) are also available for
fine-grained control over the symbol's color in different states.
